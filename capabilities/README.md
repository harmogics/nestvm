# capabilities — the seed register

The distributable declarations of this system's growable capabilities:
each seed describes one capability (atomic) or one guild (the maximal
composite) in the [FORMAT](./FORMAT.md); code never travels — declared
forms and semantic content do
([spec/guild-seeds.md](../spec/guild-seeds.md)). Lifecycle
(PROTOCOL_DEV, the cycle): while a cycle is open the seed follows its
living proposal; at close the learned detail back-propagates here, the
version bumps, and the proposal retires to `spec/history/` — the seed is
then the durable description.
