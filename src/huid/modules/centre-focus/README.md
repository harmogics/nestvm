# centre-focus — the figure lens

**Role.** The centre carousel's figure lens (`centre.focus`): one scene in
depth — the opened lid of the vault (Vol. 08 §8): heading, purpose,
barrier tally; the scene's record in three strata (ADR-005 D3), each row
a factory widget with the raw record one disclosure away; knot cards
(question, angle, grade against threshold, wound state, evidence, read
sources, return notes); the candidate panel with the acceptance moment
(`--terra`). The last extracted centre module (design_proposal §6
step 3; seed §4.4).

**Governing texts:** seed §4.1/§4.4; design_proposal §3.1, §7 (trap 4 —
the card composes factory widgets, it is not one), §12.5.1 (shared row
formers), §12.5.3 (`composer.target`); ADR-005 D3 (strata); HUID 02;
HUID 04 §5 (weight ladder; acceptance carries terra).

**Shape.** `manifest.ts` declares `consumes: [scene-detail]`, the host
keys it reads/writes and its namespaced disclosure parameters
(source menu, catalogue picks — disclosure is navigation). `view.tsx` is
a pure `select` (focus pick + ancestry from parent links) and a stateless
view. Retargeting the composer (Answer/Challenge) writes
`composer.target` — never a module-to-module call. All five gestures
commit declared decision bodies through the port.

**Interim liberties, recorded:** the catalogue (immutable package data —
a lawful derivation input, Vol. 08 §8) and the shelf (declared sources,
awaiting a session-meta/shelf contract — design_proposal §12.3.5) arrive
as host-injected props; the pre-scene state (no scene yet) renders in the
host, which still holds the session subject client-side until the strip
migrates.

**What never belongs here:** payload parsing, fetch calls, imports of
the plane, semantic computation in render, a second composer.
