# Attention matrix for the inductive interface

**Status:** Working evaluation framework

This matrix tests whether a product decision respects both the person's attention
and the Nest runtime. It is not a list of visible tabs. The seven perspectives are
working Harmogics phases: they guide movement through content.

The phases must not be conflated with the eight preconditioning threads described
in *Threads of Distinction v0.1*: Referent/Surface, Act/Object,
Fusion/Connection, Magnification/Holding, Weights/Window/Gap, Seam under Pressure,
Coordinate/Path, and Carrier/Pair. Threads are orthogonal gradients selected at
rising-cost points before or across phases. They qualify how a phase operates; they
are not descriptor kinds, source grades, or permanent UI modes. See
[cross-source-concept-analysis.md](./cross-source-concept-analysis.md).

| Perspective | Attention question | Current hold | Implementation consequence |
| --- | --- | --- | --- |
| Truth | What actually happened, and what is only interpretation? | The wave log is the single ordered truth. Raw user signals, model proposals, knot readiness, bind integrations, and human acceptance remain different records. | Every visible durable state is replayable. Model output carries provenance and never overwrites a source fact. |
| Deep | What deserves another level of inquiry now? | Depth is chosen by marking a knot, not generated everywhere in advance. | A deepened knot seeds an attributed child bind/cascade on the right; budgets and guards bound depth, fan-out, and external work. |
| Connect | Which prior meanings belong in this operation? | Non-excluded left values become candidate affinity sources, but the exact set is frozen when the bind is created. | Store explicit source refs and barrier policy. Treat the resulting provenance as a DAG, even when the document is rendered as a tree. |
| Service | What transformation is authorised, with which inputs and limits? | The user selects an allowlisted semantic operator and supplies its parameters through the single composer and compact context. | Invoke a versioned precompiled bind template. External inference is a committed intention handled by a controller, never arbitrary model-authored execution. |
| Knowledge | When does information become reusable understanding? | A signal is not knowledge merely because it was written. Reusable knowledge is a published, traceable local integration released for later affinity. | Left-side value panels point to stable integration records with evidence, uncertainty, operator, and source lineage. |
| Evolution | How can the system gain new operators without weakening its core? | New capability grows on existing rails. Runtime authoring must pass reachability, attribution, identity, and termination/budget gates. | Version operator catalogues and schemas. Treat genuinely new topology as an inert proposal and Trial until compiled and admitted. |
| Responsibility | Who chose the direction, sources, acceptance, and finish? | The person retains choice of operator, depth, source exclusion, answer, and acceptance. Never infer consent from mere text entry or from silence. |

## Cross-perspective tensions

### Silence versus truth

The UI should not narrate obvious machine state, but omission cannot erase
provenance. The resolution is asymmetric: the log remains complete while the
surface is selective. Inspection can reveal trace and evidence on demand; the
default scene stays quiet.

### Default-in affinity versus bounded context

Using every non-excluded left value respects the user's spatial model but can
silently enlarge semantic radius. The source snapshot must therefore be visible in
a compact way, bounded by policy, and inspectable before execution. The system may
warn about an excessive radius, but it must not silently remove sources or decide
their relevance after submission.

### Recursive depth versus completion

Every knot may become a child bind, so recursive possibility cannot define
success. Completion comes from the selected session's result contract, required
demands, uncertainty policy, and human acceptance. Depth is a means, not the goal.

### User control versus executable safety

The user creates a bind in product language, but current runtime safety favours a
new instance of a precompiled descriptor template. This is not a contradiction if
the UI preserves authorship of action, parameters, sources, and purpose while the
runtime preserves the closed executable function. A new function is an authoring
proposal, not an immediate invocation.

### Knowledge reuse versus premature closure

Moving a completed scene left makes it available to new affinity. It must not make
the result look unquestionably true. A value panel needs a compact uncertainty or
qualification signal and an expandable provenance path, while avoiding permanent
visual noise.

## Questions to test in implementation

- Can a person predict what one submission will do from the composer, focused
  target, operator selection, and left-source state alone?
- Can they distinguish “answer this knot” from “create an operator over this knot”
  without opening another form?
- Can they move left, right, up, and down without losing the current semantic
  position or mistaking navigation for execution?
- Can they see the exact sources of a bind at the moment the choice matters, while
  provenance stays collapsed at other times?
- Can they stop with explicit uncertainty rather than being pushed into arbitrary
  further depth?
- After replay, does the same log reconstruct the same scene, source set, child
  branches, accepted values, and final document?
- Can an evaluator identify which conclusions came from the person, the guide, a
  deterministic service, and an independent assessment?
