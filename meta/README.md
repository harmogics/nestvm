# meta — cross-level integrations

**Purpose.** This directory holds readings that gather evidence **across**
the layers — `specifications/` (the machine), `huid/` (the device),
`spec/` (the product), `src/` (the implementation), and the public
foundation (PHILOSOPHY, ECOSYSTEM) — and publish recurring principles
that belong to no single layer. A principle earns a place here only when
it is *observed* across levels, never when it is merely wished.

**Form.** Each document is a **meta-bind** (`meta-bind-NN.md`), and the
name is literal: the document instantiates the very figure it records —
it *gathers* a scope (the cited sources across levels), *integrates*
(the analysis), and *publishes* (one principle with its guards). A
meta-bind that cannot cite its gathered scope is not ready to publish.

**Authority.** Meta-binds are derived readings of the corpus and override
nothing: where a layer's own normative text disagrees, the layer governs
and the meta-bind is corrected (the ecosystem rule: shared language is
not shared authority — ECOSYSTEM, boundary 6). Corrections are appended
as dated notes; the gathered scope stays cited so the revision is
inspectable.

**Status language.** OBSERVED — evidenced in at least three distinct
levels, with the instances tabled; PROPOSED — a candidate recurrence
still gathering evidence.
